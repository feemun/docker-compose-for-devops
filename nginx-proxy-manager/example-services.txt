# 十个服务的示例配置
# 使用 create-service-config.bat 批量创建这些配置

十个服务的示例配置：

服务名称        宿主机端口    服务别名(URL路径)
user-service    3001         /users
order-service   3002         /orders
product-service 3003         /products
payment-service 3004         /payments
file-service    3005         /files
notification-service 3006    /notifications
log-service     3007         /logs
monitor-service 3008         /monitor
config-service  3009         /config
smart-calendar  8000         /smart-calendar

说明：
- 宿主机端口：后台服务在宿主机上监听的端口（如 http://localhost:3001）
- 服务别名：通过 Nginx 代理访问的 URL 路径（如 https://localhost/users）
- 代理效果：https://localhost/users -> http://localhost:3001

# 批量创建命令示例：
# 1. 复制上述服务配置（去掉注释）
# 2. 运行：create-service-config.bat batch
# 3. 粘贴服务配置
# 4. 按 Ctrl+Z 然后回车完成

# 单个创建命令示例：
# create-service-config.bat user-service 3001 /users
# create-service-config.bat order-service 3002 /orders
# create-service-config.bat product-service 3003 /products
# create-service-config.bat payment-service 3004 /payments
# create-service-config.bat file-service 3005 /files
# create-service-config.bat notification-service 3006 /notifications
# create-service-config.bat log-service 3007 /logs
# create-service-config.bat monitor-service 3008 /monitor
# create-service-config.bat config-service 3009 /config

访问地址示例：
https://localhost/users          -> http://localhost:3001 (user-service)
https://localhost/orders         -> http://localhost:3002 (order-service)
https://localhost/products       -> http://localhost:3003 (product-service)
https://localhost/payments       -> http://localhost:3004 (payment-service)
https://localhost/files          -> http://localhost:3005 (file-service)
https://localhost/notifications  -> http://localhost:3006 (notification-service)
https://localhost/logs           -> http://localhost:3007 (log-service)
https://localhost/monitor        -> http://localhost:3008 (monitor-service)
https://localhost/config         -> http://localhost:3009 (config-service)
https://localhost/smart-calendar -> http://localhost:8000 (smart-calendar)
